객체 (Object):
객체는 데이터와 그 데이터를 처리하는 함수를 하나로 묶은 개념이다. 객체는 **상태(state)**와 **행동(behavior)**을 가진다. 상태는 객체의 속성(예: 변수)이고, 행동은 객체가 수행할 수 있는 함수(예: 메서드)이다. 객체는 실세계의 개체를 모델링한 것이다.
클래스 (Class):
클래스는 객체를 생성하기 위한 설계도 또는 틀이다. 클래스는 객체의 속성과 메서드를 정의하며, 이 클래스를 바탕으로 여러 개의 객체를 만들 수 있다. 클래스 자체는 객체가 아니며, 객체를 생성하기 위한 청사진 역할을 한다.
인스턴스 (Instance):
클래스에서 실제 객체를 생성하면 그 객체를 인스턴스라고 부른다. 클래스는 객체의 설계도일 뿐, 실제로 메모리 상에 존재하는 것은 인스턴스이다. 즉, 클래스로부터 생성된 객체가 인스턴스이다.
추상화 (Abstraction):
추상화는 객체의 중요한 속성이나 기능만을 노출하고, 복잡한 내부 구현은 숨기는 개념이다. 추상화는 사용자가 시스템을 쉽게 이해하고 사용할 수 있도록 도와준다. 예를 들어, 자동차를 운전할 때 핸들, 가속기, 브레이크만 사용하지만, 자동차 엔진과 같은 내부 구조는 보지 않고 사용한다.
상속 (Inheritance):
상속은 기존 클래스를 기반으로 새로운 클래스를 만들 수 있게 해주는 특성이다. 이를 통해 코드의 재사용성을 높이고, 공통적인 특성을 부모 클래스에서 정의하고 자식 클래스에서 이를 상속받아 사용할 수 있다. 예를 들어, 동물 클래스를 부모 클래스로 하고, 고양이, 강아지와 같은 자식 클래스가 있을 수 있다. 자식 클래스는 부모 클래스의 속성과 메서드를 그대로 상속받을 수 있다.
캡슐화 (Encapsulation):
캡슐화는 객체의 내부 데이터를 숨기고 외부에서는 그 데이터에 접근할 수 없도록 만드는 개념이다. 이를 통해 객체의 상태를 안전하게 보호하고, 객체가 일관성 있게 동작하도록 할 수 있다. 예를 들어, 객체의 필드는 private로 설정하고, 외부에서 이를 조작하려면 getter와 setter 메서드를 사용하게 할 수 있다.
다형성 (Polymorphism):
다형성은 하나의 인터페이스나 메서드가 여러 형태를 가질 수 있게 하는 개념이다. 즉, 같은 이름의 메서드가 서로 다른 객체에서 다르게 동작하도록 만드는 것이다. 이는 코드의 유연성을 높이고, 다양한 객체를 동일한 방식으로 처리할 수 있도록 도와준다. 예를 들어, 동물 클래스의 makeSound() 메서드가 고양이 클래스에서는 "야옹"을, 강아지 클래스에서는 "멍멍"을 출력하는 방식이다.
메시지 (Message):
객체는 다른 객체와 상호작용을 할 때 메시지를 주고받는다. 객체는 특정 메서드를 호출하거나 데이터를 전달함으로써 다른 객체와 소통한다. 객체 간의 상호작용은 이 메시지를 통해 이루어진다.
인터페이스 (Interface):
인터페이스는 클래스가 반드시 구현해야 할 메서드의 계약을 정의한다. 즉, 인터페이스는 클래스가 구현해야 하는 메서드들의 서명을 제공하고, 이를 통해 여러 클래스가 동일한 메서드를 다르게 구현할 수 있다. 인터페이스는 다형성 구현에 중요한 역할을 하며, 특정 기능을 요구하는 클래스들이 공통된 메서드를 구현하게 한다.
